PEGasuS: 6D Pose Estimation from a Single Image and a 3D Gaussian Splatting Model

Matteo Bortolon1,2,3, Theodore Tsesmelis1, Stuart James1,4, Fabio Poiesi2, Alessio Del Bue1,
1, 2, 3, 4
arXiv Video

PEGasuS leverage the rays projected outwards from the camera optical center reaching the surfaces of the Quadrics (i.e. Ellipsoids) establishing a correspondence between image pixels and 3DGS Quadrics. These rays-pixels correspondences are used by our approach to estimate the camera pose from a target image.

Abstract

3D Gaussian Splatting (3DGS) is a Novel View Synthesis (NVS) approach that can render accurate and fast photorealistic images from arbitrary viewpoints. In this work, we show how 3DGS models can be used to recover the camera pose of never seen target views.

We introduce a novel radiant Quadricell sampling technique to cast a set of rays equally distributed on the surface of each 3D Gaussian modelled as a Quadric. Then, we assign rays to the pixels of the target image to synthesize their color values. We compute this assignment by using a learned attention map, which scores the best ray-pixel correspondences. Once rays and pixels are matched, the camera pose can be recovered by solving a weighted Least Squares problem.

PEGasuS needs no initial pose guess, as opposed to existing pose estimation NVS methods that instead need it as prior (e.g., iNeRF). Moreover, compared to the existing NVS baselines for pose estimation, PEGasuS can improve the angular accuracy by 41% for synthetic and 70% for real experiments. Our method operates in real-time, reaching 34 fps on consumer hardware.

Method

PEGasuS estimates the camera pose, given a target image and a pre-computed 3DGS model quadrics. We use the set of quadrics in the 3DGS model and their geometrical parametrization to fit a radiant Quadricell on a given quadric surface, which divides the quadric surface into cells of approximately equal area (A). We then cast a set of rays, using the radiant quadricell. A subset of these rays would be expected to intersect the camera, as in the case of the Quadric generation in 3DGS. We identify the subset by an attention map between the embeddings of the target image and the generated rays (C). The goal of the attention map is to select the rays that best correlate with the image by scoring them using the embedding similarity. The scores provide a subset of candidate rays which are likely to fall within the image plane and match the encoded image pixel properties (B). Finally, we recover the pose by minimizing the sum of perpendicular distances from the unique solution point to all the lines, i.e., the point which is closest to all the lines, by using the weighted Least Squares algorithm (D). The direction of the selected rays also allows us to compute the yaw and pitch of the camera (x and y-axis) while the roll is computed based on the ray-image pixel association (z-axis).
